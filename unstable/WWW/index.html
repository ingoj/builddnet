unstable.buildd.net

Welcome to a Debian autobuild system frontend.

These pages are intended to show additional information to http://buildd.debian.org or more exactly it is basically the same information, but shown in a different, maybe better readable form.
These pages originally were created for m68k, but are now available for other archs as well:

unstable: [alpha] [amd64] [arm] [hppa] [i386] [ia64] [m68k] [mips] [mipsel] [powerpc] [s390] [sparc]


Package status (all archs)


Some more information about the buildd system can be found at:

  • Please visit the www.buildd.net FAQ here.
  • See here to see when the buildds update their status.
  • An explanation how a buildd works, posted by Michael Schmitz on the m68k-build list.
  • Some common build problems.
  • Graphs/stats and build logs for all arches are at buildd.debian.org.
  • http://people.debian.org/~wouter/wanna-build-states by Wouter Verhelst
  • Graphical stats. for Needs-Build over all archs (gnuplot script by Goswin Brederlow).
  • http://bjorn.haxx.se/debian/ is a nice page to find out, what's going into testing and what not - and why not.
  • A conceptual draft on Multibuild- Multibuild is intended to be a replacement for the current buildd system.

Note:

When you contact the porters list or a buildd admin, please do mention which buildd you're talking about. The buildd name is on top of each buildd log mail. See: http://buildd.debian.org.

Currently not all information on the archs pages are up to date, i.e. the list of listed autobuilder machines and the contact addresses for the arch. Please send in additional information about specific autobuilders and contact addresses to me (ij a 2004,bluespice,org).
